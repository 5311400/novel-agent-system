# Skill：Token 高效子 Agent 委派协议（强制协议）

## 1. 委派决策树
- **必须委派**：涉及 >3 个独立实体的信息检索、大规模设定补全
- **禁止委派**：单一句子润色、简单逻辑衔接、已知上下文内的微小修改
- **并行豁免**：仅信息收集类可并行；所有生成类强制串行

## 2. 三明治压缩法
向子 Agent 传递指令时，必须使用以下模板，控制总 Prompt 在 300 Token 以内：
```
【必须上下文】(50词内) 当前章节梗概 + 前一章结尾最后一句话。
【唯一任务】(30词内) 具体要产出的东西，量化。
【硬性约束】(50词内) 严格遵守世界观文档第X条 + 角色当前情绪状态。
```
**反例**：禁止传递整本前10章全文给子Agent。

## 3. 子Agent输出极简归档格式
```
[核心产出]：正文/列表（允许长文本）
[变更摘要]：仅列出本次修改了哪些设定文件
[待办提醒]：下一章必须解决的伏笔（不超过2条，每条15字内）
```

## 4. 上下文续写策略
写作Agent写下一章时，总控**只传递**以下3个片段：
① 上一章结尾的 200 字
② 当前章节的 3 个关键节拍点
③ 正在对话的 2 个角色的最新性格标签（各50字）

## 5. 失败熔断与降级
如果子Agent返回内容 Token 数超过预期，总控强制执行 truncate_and_rewrite（截断并重写摘要），绝不将冗余内容原样纳入知识库。
