# 小说 Agent 系统 — 投喂说明

> 以下是我正在使用的多 Agent 小说写作系统的完整配置。
> 请基于此配置理解我的 Agent 结构、分工和工作流，后续我会按角色加载对应 Skill 文件或直接按这套规则协作。

---

## 系统概览

Skill 定流程 + Agent 干专活 + Artifact 管流转 + Data 保一致

### Agent 角色（6 个）

| Agent | 模型 | 职责 |
|-------|------|------|
| **Coordinator（总控）** | 最强模型 | 任务分解、状态机调度、冲突仲裁、原子锁管理 |
| **WorldBuilder（世界观）** | 轻量模型 | 地理/势力/力量体系/规则档案的查询和更新 |
| **CharacterKeeper（人设）** | 轻量模型 | 角色档案维护、OOC 校验、跨章节连续性检查 |
| **ChapterWriter（主笔）** | 最强模型 | 按节拍表起草章节正文，遵守首尾锚点 |
| **StyleGuard（文风）** | 轻量模型 | AI腔检测、7:2:1文风配比、句长/对话密度/环境检查 |
| **PolishEditor（质检）** | 最强模型 | 战力/OOC/设定/逻辑错误分级、深度质检报告 |

### 工作流（写一章）

```
ChapterWriter 完稿
    ↓
StyleGuard 文风报检（24种AI模式检测 + 7:2:1配比检查）
    ↓
Coordinator 决策（通过/打回）
    ↓
PolishEditor 深度质检（三级错误分级 + 跨档案一致性）
    ↓
Coordinator 仲裁 → 归档 data/chapters/
```

### 关键规则

- **写前执行 /init**：检测世界观和人物档案是否就绪，未就绪不得进入起草
- **首尾锚点**：章节开头必须回上一章悬念，结尾必须对齐节拍表最后一拍
- **AI腔宽松检测**：500字滑动窗口≥3次才标记，战斗场景放宽到5次
- **重度失败暂停**：子Agent重试2次失败→暂停等用户，禁止总控代写
- **写作Agent不新增设定**：ChapterWriter 禁止在正文中引入新世界观或人物关系
- **版本校验**：所有产出绑定 Snapshot-GV，过期驳回重跑

### 文风配比（7:2:1）
- 70% 爽文快节奏（短句≤20字、对话密、钩子结尾）
- 20% 热血战斗（动作短促、爆发感强）
- ≤10% 古风点缀（单段环境≤200字）

### 文件结构简版
```
data/
├── world/          # 世界观档案（geography, factions, power-system, rules）
├── characters/     # 人物档案（protagonist, main-cast, supporting）
├── chapters/       # 已发布章节终稿
├── INDEX.md        # 全局版本号 GV
├── plot-log.md     # 剧情摘要
├── project-board.md# 进度看板
└── changelog.md    # 变更日志
```

### 常见指令
- `/init` — 初始化系统（首次使用）
- `/write-chapter N` — 写第 N 章
- `/status` — 查看项目进度
- `/patch-review` — 查看质检进化补丁
