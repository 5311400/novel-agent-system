# 小说源文本 → 结构化知识库 工作流程

## 你要做的
把小说每章/每几章的原文文件放进 `小说源文本/` 文件夹

## 我来做的（Codex / AI）
读取原文 → 逐章输出结构化分析 → 写入 `知识库/章节分析/`

## 分析持续积累
每分析一本，结果追加到知识库
分析完 10-20 本后，自动生成跨书统计：常见开篇模式、黄金三章规律、节拍通用模型

---

## 文件命名规则

输入（小说源文本/）：
```
小说源文本/
├── 斗破苍穹_第1章.txt
├── 斗破苍穹_第2章.txt
├── 凡人修仙传_第1章.txt
└── ...
```

输出（知识库/章节分析/）：
```
知识库/章节分析/
├── 分析模板.md                    ← 本文件，定义分析格式
├── 斗破苍穹_分析.md               ← 分析结果
├── 凡人修仙传_分析.md
├── 跨书汇总_开篇模式.md           ← 10本后自动生成
├── 跨书汇总_节拍通用模型.md
└── ...
```

## 启动方法
把原文放进 `小说源文本/` 后，跟我说一声，我立刻开始逐个分析。
