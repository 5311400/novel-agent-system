## 二、Skill 直接可用的硬规则（从知识库提炼）

### 通用规则（跨所有 Skill）

```
- 每章必须有价值转换（正→负 或 负→正）      ← 麦基
- 每章至少 1 个小冲突点                      ← 冲突与悬念
- 爽点间隔 ≤5 章（压抑期 ≤7 章）            ← 跨书汇总
- 前200字必须建立钩子                        ← 跨书汇总
- 每6-8章一个大爽点                          ← 跨书汇总
```

### ChapterWriter 规则

```
- 参照 15 节拍表设计每章节拍                 ← 救猫咪
- 开篇模式可选：场景先行 / 冲突先行 / 日常先行 / 悬念先行 ← 跨书汇总
- 前30章节奏按 5 种模型选型                   ← 跨书汇总
- 卡文时从 36 种冲突模式中选一个              ← 小说课2
```

### StyleGuard 规则

```
- 检查 7:2:1 文风配比                        ← 跨书汇总 + 自定义
- 500字窗口检测 24 种 AI 腔模式              ← 自定义
- 勒古恩语言规则：句长多样 / 动词优先 / 副词节制 ← 写小说最重要的十件事
```

### PolishEditor 规则

```
- 场景价值转换检查（没有变化=无用章节）        ← 麦基
- 英雄之旅阶段校验（第N章应处于哪个阶段）      ← 千面英雄
- 冲突类型一致性检查（不可切换冲突类型）        ← 跨书汇总
```

在下个迭代中，可以将以下规则直接编码进 `tools/validate.py`：

- [ ] LOCK 系统检查（主角/目标/对抗/结尾）
- [ ] 15节拍表进度检查
- [ ] 前30章节奏模型匹配
- [ ] 开篇钩子类型检测
- [ ] 冲突密度统计（冲突/字数比）
- [ ] 爽点间隔监控

> 每次写入一章后，validate.py 可运行上述检查，输出质检报告。
